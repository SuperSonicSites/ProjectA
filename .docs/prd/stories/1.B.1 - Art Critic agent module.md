# 1.B.1 — Art Critic agent module (QA)

Epic: `1 - PaperPause Autonomy System`

## Goal
Create a vision QA agent that returns structured, validated output for coloring-page quality.

## Scope
- New agent under `.agents/art-critic/` (name can vary, but must follow `.agents/` conventions).
- Input: image URL (or CF variant), optional context (category/collection, prompt).
- Output: strict JSON object:
  - `qa_result`: `pass` | `fail`
  - `reason`: bounded enum (e.g., `TEXT_PRESENT`, `GRAYSCALE_SHADING`, `BLURRY_LINES`, `OPEN_PATHS`, `CANVAS_OCCUPANCY_LOW`, etc.)
  - `reason_details`: short string
- **Agent Requirements**:
  - Model: `gemini-3-flash-preview`
  - Thinking Level: **MEDIUM**
  - Vision Resolution: `MEDIA_RESOLUTION_ULTRA_HIGH`

## Acceptance criteria
- Output is machine-parseable and validated.
- Reasons are from a bounded enum (no free-form “reason” strings).
- The agent can operate in both modes:
  - `observe` (report only)
  - `enforce_failfast` (used by orchestrator; agent itself still just evaluates)

## Rollback
- Disable QA at orchestration layer (`ENABLE_QA=0`).
