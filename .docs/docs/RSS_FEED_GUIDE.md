## RSS Feed Content Structure

When a post is published (`draft: false`), it appears in RSS as:

```xml
<item>
  <title>Cat Coloring Page: Gardener Cat watering a</title>
  <link>https://paperpause.app/animals/cats/gardener-cat/</link>
  <pubDate>Sun, 15 Dec 2024 02:13:58 -0700</pubDate>
  <description>A beautiful cat coloring page generated by AI...</description>
  
  <!-- Pinterest will use this image (1000x1500px) -->
  <enclosure url="https://...r2.cloudflarestorage.com/cats/cat-123.png" />
  <media:content 
    url="https://...r2.cloudflarestorage.com/cats/cat-123.png" 
    width="1000" 
    height="1500" 
  />
</item>
```

Pinterest reads this and automatically creates a Pin with:
- **Image**: 1000x1500px from `<media:content>`
- **Title**: From `<title>`
- **Description**: From `<description>`
- **Link**: From `<link>` (drives traffic back to your site)

## Troubleshooting

### "No items in RSS feed"

**Cause**: All posts have `draft: true`  
**Fix**: Approve posts in dashboard or set `draft: false`

### "Pinterest says feed is empty"

**Cause**: RSS feed not updated after publishing  
**Fix**: Run `npm run build` and redeploy

### "Images not showing in RSS reader"

**Cause**: Image URLs might be incorrect or CF Images not configured  
**Fix**: Check that R2 image URLs are publicly accessible

### "Pinterest says links don't match claimed domain"

**Cause**: Links point to wrong domain  
**Fix**: Verify `baseURL` in `hugo.toml` is `https://paperpause.app/`
